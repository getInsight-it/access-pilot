# Segurança

## Reportando vulnerabilidades ou problemas de segurança

Se você encontrou uma falha de segurança ou vulnerabilidade no Access Pilot ou nas SDKs adjascentes, você deve enviar um e-mail para `contato[@]getinsight.it`.

Por favor, inclua o máximo de informação pertinente no seu e-mail, de forma a nos ajudar a entender e resolver o problema:

- Onde está o problema?
- Tipo de problema (ex: injeção de SQL ou de scripts cross-site, falhas de autorização, etc)
- Se possível, caminho completo ou links para os arquivos de código-fonte onde o problema está
- Passo-a-passo de como reproduzir o problema
- Prova de conceito de exploração da vulnerabildiade, se viável

Caso você precise enviar informações confidenciais, por favor nos contactar para que possamos trocar chaves PGP.

## Atenção

A não ser onde disposto em contrário, todo código de exemplo nos nossos repositórios não tem qualquer garantia (nem mesmo de segurança). Código de exemplo pode conter vulnerabilidades e, portanto, qualquer código de exemplo não deve ser usado de forma literal em ambiente produtivo. O código de exemplo existe somente com o propósito ilustrativo e didático.
